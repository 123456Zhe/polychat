# PolyChat 新功能实现计划

## 功能1：修复封禁无效

### 问题
`isUserBanned()` 只在登录时调用（server.mjs:439），已登录用户被封禁后仍可正常使用所有功能。

### 修改方案

**server.mjs:**
1. **`requireUser()` (line 310)** — 在返回 user 前添加封禁检查：
   ```js
   function requireUser(req, res) {
     const user = currentUser(req);
     if (!user) return sendError(res, 401, '未登录');
     if (isUserBanned(user)) return sendError(res, 403, '账号已被封禁');
     return user;
   }
   ```
2. **WebSocket 升级 (line 1052)** — `currentUser()` 返回后添加封禁检查：
   ```js
   const user = currentUser(req);
   if (!user) { socket.write('HTTP/1.1 401...'); return socket.destroy(); }
   if (isUserBanned(user)) { socket.write('HTTP/1.1 403...'); return socket.destroy(); }
   ```
3. **消息发送 (line 930)** — 在 mute 检查前添加 ban 检查：
   ```js
   if (isUserBanned(user)) return json(res, 403, { error: '账号已被封禁' });
   if (isUserMuted(user)) return json(res, 403, { error: '你已被禁言' ... });
   ```

**无需修改 GUI/TUI** — 服务端统一拒绝后客户端会收到错误。

## 功能3：房间公告与讨论区

### 数据库

```sql
CREATE TABLE IF NOT EXISTS room_announcements (
  id INTEGER PRIMARY KEY,
  room_id INTEGER NOT NULL REFERENCES rooms(id) ON DELETE CASCADE,
  author_id INTEGER NOT NULL REFERENCES users(id),
  title TEXT NOT NULL,
  content TEXT NOT NULL,
  is_active INTEGER NOT NULL DEFAULT 1,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT
);
CREATE INDEX IF NOT EXISTS idx_announcements_room ON room_announcements(room_id, is_active);
```

### API 端点

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/rooms/:id/announcements` | 获取当前公告 | 房间成员 |
| POST | `/api/rooms/:id/announcements` | 发布公告 | 房间管理员 |
| PUT | `/api/rooms/:id/announcements/:aid` | 编辑公告 | 作者/管理员 |
| DELETE | `/api/rooms/:id/announcements/:aid` | 删除公告 | 管理员 |

### 讨论区
- 公告下的讨论使用 `messages` 表，设置 `thread_root = announcement_id`
- 获取公告讨论：`GET /api/rooms/:id/announcements/:aid/messages`
- 发布讨论回复：`POST /api/rooms/:id/announcements/:aid/messages`（自动设置 thread_root）

### Web UI

在置顶消息区域下方添加公告区域：
```html
<div v-if="activeAnnouncement" class="announcement-banner">
  <div class="announcement-header">
    <span>📢 公告</span>
    <strong>{{ activeAnnouncement.title }}</strong>
    <button @click="showAnnouncementDiscussion = !showAnnouncementDiscussion">
      讨论 ({{ announcementDiscussionCount }})
    </button>
  </div>
  <div class="markdown" v-html="markdown(activeAnnouncement.content)"></div>
  <div v-if="showAnnouncementDiscussion" class="announcement-discussion">
    <!-- 讨论消息列表 -->
  </div>
</div>
```

## 功能4：扩展邀请方式

### 数据库

```sql
CREATE TABLE IF NOT EXISTS room_invite_codes (
  id INTEGER PRIMARY KEY,
  room_id INTEGER NOT NULL REFERENCES rooms(id) ON DELETE CASCADE,
  code TEXT NOT NULL UNIQUE,
  type TEXT NOT NULL CHECK(type IN ('one_time', 'timed', 'permanent')),
  created_by INTEGER NOT NULL REFERENCES users(id),
  max_uses INTEGER,
  use_count INTEGER NOT NULL DEFAULT 0,
  expires_at INTEGER,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP
);
CREATE INDEX IF NOT EXISTS idx_invite_codes_room ON room_invite_codes(room_id);
CREATE UNIQUE INDEX IF NOT EXISTS idx_invite_codes_code ON room_invite_codes(code);
```

### API 端点

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| POST | `/api/rooms/:id/invite-codes` | 创建邀请码 | 房间管理员 |
| GET | `/api/rooms/:id/invite-codes` | 列出邀请码 | 房间管理员 |
| DELETE | `/api/rooms/:id/invite-codes/:cid` | 撤销邀请码 | 房间管理员 |
| POST | `/api/rooms/join` | 使用邀请码加入 | 所有用户 |

### 邀请码生成
```js
function generateCode() {
  return crypto.randomBytes(8).toString('base64url'); // 11字符
}
```

### 创建邀请码逻辑
```js
// POST /api/rooms/:id/invite-codes
// body: { type: 'one_time'|'timed'|'permanent', max_uses?, hours? }
const code = generateCode();
// one_time: max_uses=1
// timed: expires_at = Date.now() + hours*3600_000
// permanent: 无限制
```

### Web UI

邀请弹窗添加两个 Tab：
1. **用户列表** — 显示所有用户，点击邀请
2. **邀请码** — 创建/管理邀请码，复制链接

```html
<div class="invite-tabs">
  <button @click="inviteTab='users'" :class="{active: inviteTab==='users'}">用户列表</button>
  <button @click="inviteTab='codes'" :class="{active: inviteTab==='codes'}">邀请码</button>
</div>
```
